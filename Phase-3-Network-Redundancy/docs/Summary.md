Phase 3 focused on transforming the network from a functional routed campus into a highly available enterprise design by implementing redundancy at both Layer 2 and Layer 3. The existing topology was expanded by converting the secondary distribution switch into a Layer 3 switch, adding redundant uplinks from the access layer, and deploying a Layer 3 EtherChannel between the distribution switches.

Gateway redundancy was implemented using HSRP to provide highly available default gateways for all VLANs, while OSPF was expanded to dynamically exchange routes between both distribution switches and the WAN routers. Rapid PVST+ was configured and optimized to align spanning tree root bridge placement with HSRP active gateways, ensuring efficient traffic forwarding while preventing Layer 2 loops.

Throughout implementation, several real-world issues were encountered and resolved, including Layer 3 EtherChannel configuration sequencing, HSRP neighbor formation, and upgrading DSW2 from an IP Base to an IP Services IOS image to enable OSPF support. The IOS upgrade was completed by configuring a TFTP server and transferring the new image between switches, providing valuable experience with Cisco software maintenance and troubleshooting.

The completed design was validated through extensive failover testing. Distribution switch failures, EtherChannel member failures, uplink failures, and Layer 3 EtherChannel failures were simulated to verify gateway redundancy, dynamic routing convergence, spanning tree reconvergence, and link resiliency. All planned test scenarios produced the expected results, demonstrating that the network maintained availability and recovered successfully during simulated failures.

By the end of Phase 3, the lab had evolved from a basic routed campus network into a resilient enterprise campus capable of maintaining service during device and link failures while automatically reconverging through dynamic routing and first-hop redundancy protocols.
